import React from "react"

const PersonalInformationCard = () => {
  return (
    <div>PersonalInformationCard</div>
  )
}

export default PersonalInformationCard

Import it like this:

import PersonalInformationCard from "@/components/dashboard/profile/personal-information-card"

Notice no curly braces.

Pros
Slightly less typing when importing.
VS Code often auto-imports it nicely.
Cons
You can rename it during import without realizing it:
import Card from "@/components/dashboard/profile/personal-information-card"

or

import Test from "@/components/dashboard/profile/personal-information-card"

Both work, which can make a large codebase less consistent.

Option 2: Named Export
export function PersonalInformationCard() {
  return (
    <div>PersonalInformationCard</div>
  )
}

Import it like this:

import { PersonalInformationCard } from "@/components/dashboard/profile/personal-information-card"

Notice the curly braces.

Pros
The component name is always consistent.
Easier to refactor across a large project.
Better autocomplete.
Better compile-time errors.
Allows multiple exports from the same file if needed.
